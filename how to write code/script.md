写代码的目的：打比赛；和计算机交流——让计算机帮我们做一些事情；创造一些东西——比如游戏、软件、网站等；解决问题——比如数据分析、自动化任务等

为什么要写好的代码：运行效率、减少错误率——清晰、结构化的代码更容易调试和维护；团队合作——在团队中，好的代码可以让其他人更容易理解和使用；

学习写代码：搜索、AI、实践；模仿、阅读优秀的代码；审阅自己的代码（让ai给出改进意见）

------

考场写代码：先读题、计算样例确保题意正确；思考解法；思考代码结构；写代码。最后一步可以边想边写，前面需要先想好

写代码的习惯：命名规范、代码结构。具体代码格式倒是不太重要，因为可以自动化改好。

代码的编译运行过程。冯诺依曼架构中，程序也是数据的一种。控制器读取存储器中的指令来执行程序。机器底层执行时使用机器语言，因此我们写的代码实际上是先被翻译成机器语言存储到存储器中，供我们后续执行。进行这个翻译的也是一个程序——编译器（对于 C++ 而言，对于其他语言还有解释器等）。
