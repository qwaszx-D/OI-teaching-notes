# 字符串讲义第三轮审阅建议

审阅范围：当前目录下所有 `.tex` 文件。本轮按你的要求忽略前两轮审阅中已经给出的条目；下面只列这次重新阅读时额外发现的问题或可读性风险。

符号表：`introduction.tex` 中已有基础符号表，本轮没有再改动。

## `chapter2.tex`：Border、周期与字符串匹配

### 应修正

- 第 19、127、141 行仍直接使用了 $n$，但本章当前没有统一写明“令 $n=|s|$”。建议在本章开头定义一次，或把这些位置改成 $|s|$。
- 第 47 行“计算所有 border”的递推默认 $j>0$。如果把空 border 也计入，$s[j]$、$s[b+1]$ 在 $j=0$ 时没有定义。建议写成“对非空 border 有……”并单独说明空 border。
- 第 133 行抽象命题 $P(x)$ 的闭包条件应限制在定义域内：如果 $P(x)$ 成立，则 $P(kx)$ 对所有满足 $kx\le M$ 的 $k$ 成立。

## `chapter3.tex`：有限状态自动机

### 应修正

- 第 123 行 DFA 转正则表达式的状态消除证明少了自环处理。消去状态 $q$ 时，$u\to v$ 的新转移应包含 $R_{u q}(R_{q q})^*R_{q v}$，不只是简单缩掉所有 $u\to q\to v$。
- 第 166 行限制长度等价关系的性质少了空串条件。应为：$x\equiv_L^{k+1}y$ 当且仅当 $x\equiv_L^0y$ 且对任何 $c\in\Sigma$ 都有 $xc\equiv_L^k yc$。第 175 行证明中也用了这个不完整的等价式，需要同步修正。

### 建议澄清

- 第 83 行 $O(\sum_i s_i\text{ 涉及的转移边数})$ 中的 $s_i$ 像是字符但又参与求和，表意不清。建议改成“对每个读入字符 $c$，枚举当前状态集合经 $c$ 标号边能到达的状态”之类的描述。

## `chapter4.tex`：Trie 和 AC 自动机

### 建议澄清

- 第 73 行 `AHO-Corasick` 通常写作 `Aho-Corasick`。这不是算法错误，但建议统一标准拼写。

## `chapter5.tex`：后缀结构

### 应修正

- 第 62 行 Ukkonen 代码在判断 `!v` 之前先计算了 `int c{s[start[v] + rem - 1]};`。当 `v==0` 时会读 `start[0]` 和 `s[...]`，依赖未说明的哨兵/默认值。建议把 `c` 的计算移到确认 `v` 存在之后。
- 第 305 行推导最后出现了 $h(|t|-1)$，但本段没有定义 $h$。建议在差分后显式定义
  $$
  h(d)=\sum_{k+l-2=d}\hat f(k)\hat g(l).
  $$

### 建议澄清

- 第 105 行 “[TJOI2015] 弦论” 的条目写成“求字典序第 $k$ 小的本质不同子串和子串”，这里“和子串”不太通顺。建议改成“本质不同子串/可重复子串”或拆成两种版本。
- 第 125 行 LCP 公式默认 $i<j$。建议写明若 $\mathtt{rk}$ 顺序相反则先交换；若两个后缀相同则 LCP 为后缀长度。

## `chapter6.tex`：回文结构

### 建议澄清

- 第 40 行 NOI2023 解法中使用 $s[i+l+h]$ 和 $s[i+l-1-h]$ 比较最大回文半径外侧字符。若回文已经扩展到字符串边界，需要说明使用哨兵或单独处理边界，否则实现时容易越界。
