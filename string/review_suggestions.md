# 字符串讲义审阅建议

审阅范围：当前目录下所有 `.tex` 文件。

已直接修改：`introduction.tex` 已改为“字符串基础与符号约定”，`string.tex` 已在目录后、第一章前引入该文件。

## 总体建议

- 建议全文统一说明“数学叙述默认 1 下标、代码可用 0 下标、`[l,r)` 为半开区间”。已在新增的基础符号页中补充，但各章中切换下标的位置仍建议局部提醒。
- 多处高级结论直接给出证明草图，适合竞赛训练，但对第一次学习的读者较跳。建议对 KMP/fail 树、AC 自动机、SAM、回文树各补一个小例子或图。
- 章节中有若干 `TODO` 或“暂时不会”。如果这是正式发给学生的版本，建议隐藏未完成章节，或统一标成“拓展阅读/待补”。
- 代码片段大多偏模板，但有些缺少初始化、根节点约定和边界条件。建议正文明确“伪代码”还是“可直接提交模板”。

## `chapter1.tex`：字符串哈希

### 应修正

- 第 15 行与第 20 行对随机参数 $x$ 的范围不一致：前面定义 $1\le x<p$，后面写随机选择 $0\le x<p$。建议统一为 $x\in \mathbb F_p$ 或 $x\in \mathbb F_p^\*$，并相应调整碰撞概率分母。
- 第 34--38 行碰撞概率证明建议补一句：比较不同长度字符串时，可以把短串对应多项式前面补零；由于字符编码不含 $0$，两个不同字符串得到的多项式差不是零多项式。
- 第 40 行并集界可能大于 $1$。严格表述应为“不超过 $\min\{1,\frac{n(n-1)(l-1)}{2p}\}$”，或说明这是 union bound。
- 第 44 行“双哈希”应强调两个哈希函数需要独立，例如独立模数或独立随机参数；碰撞概率更严谨地写成约 $\left(\frac{l-1}{p}\right)^2$。

### 建议澄清

- 第 51--52 行子串哈希公式切换到了 1 下标，而前面定义用 $i=0,\ldots,|S|-1$。建议在公式前写明这里改用 1 下标，并定义前缀哈希 $H_i=f(S[1,i])$。
- “$O(1)$ 判断字符串是否回文”需要同时维护正串和反串哈希；建议在应用列表后补一句。
- “最长公共子串 $O(n\log n)$”通常还需要二分长度并用哈希集合/排序判重，建议写出一句方法概述。

## `chapter2.tex`：Border、周期与字符串匹配

### 应修正

- 第 55 行“如果 $p,q$ 都是周期，那么 $p+q$ 也是周期”需要前提 $p+q\le |s|$，除非全文允许大于长度的 vacuous period。第 58 行的 $kp$ 同理需要 $kp\le |s|$。
- 第 67 行弱周期引理证明中“对任何 $i\le p\le |s|-q$”读起来像一个条件链，建议改成“对 $1\le i\le p$，由 $p+q\le |s|$ 得 $i\le |s|-q$，因此……”。
- 第 153--168 行关于 $l[i]$ 的引理中，若 $j=l[i]$，则 $s[1,j]$ 是 $s[1,l[i]]$ 自身，不是前文定义的 proper border。建议改为“border（允许自身）”，或写成“$j=l[i]$ 或 $s[1,j]$ 是 $s[1,l[i]]$ 的 border”。
- 第 230 行 Z 函数代码直接写 `z[2]=0`，当 $n=1$ 时越界。建议删除该行或加条件。
- 第 236 行用 Z 数组计算 $f$ 时，应说明枚举 $i$ 从 $2$ 开始；如果从 $1$ 开始，`z[1]=n` 会把每个前缀自身当成 border。

### 建议澄清

- 第 70--77 行周期引理证明很压缩。建议补充“互质时边 $(i,i+q\bmod p)$ 构成一个环；删去一条边仍连通，因此前 $p$ 个位置相等，再由周期性推出全串相等”。
- 第 82 行“反过来”建议写明 $x$ 是正整数，且由 $x+p\le |s|$ 可知 $x\le |s|$。
- 第 85 行“$s$ 的长度不超过 $|s|/2$ 的所有周期”建议改成“$s$ 的所有不超过 $|s|/2$ 的周期”。
- 第 224 行 `\begin{descbox}[Z 算法/扩展 KMP]` 虽然当前模板可编译，但与前文 `descbox` 用法不同；建议统一成 `\begin{descbox}{算法}[Z 算法/扩展 KMP]`。

## `chapter3.tex`：有限状态自动机

### 应修正

- 第 11--15 行列出的语言例子包含非正则语言、上下文无关语言、不可判定语言。建议在例子后立刻注明“这些只是语言的例子，不都能被有限自动机识别”。
- 第 30 行 DFA 定义中写“字符串 $s$ 在 NFA $M$ 上的处理过程”，应为 DFA。
- 第 69--78 行 NFA 定义允许 $\epsilon$ 转移，但接受条件只按输入字符 $s_i$ 走，未处理任意数量的 $\epsilon$ 转移。建议改成“存在一条路径，其边标号拼接后等于 $s$”，并定义 $\epsilon$-closure。
- 第 83--88 行 NFA 转 DFA 的子集构造在有 $\epsilon$ 转移时需要先取 $\epsilon$-closure。
- 第 96 行“任意字符 $c\in\Sigma_\epsilon$”不准确，$\epsilon$ 不是字符。建议拆成“任意 $c\in\Sigma$”和“$\epsilon$ 表示语言 $\{\epsilon\}$”。
- 第 107 行 `$R_1{n,m}$` 在数学模式下排版不清，建议写成 `$R_1^{\{n,m\}}$` 或用 `\verb|R{n,m}|` 说明工程语法。
- 第 129 行“交差并补”建议改成“交、差、并、补”。
- 第 195、197 行 `Hopcraft` 应为 `Hopcroft`。

### 建议澄清

- 第 80 行 NFA 识别复杂度需要说明是否已经消除了 $\epsilon$ 转移，或是否预处理了 $\epsilon$-closure。
- 第 113 行“某些扩展可能使得正则表达式的判定问题变得 NP-hard”过于笼统。建议举例说明是带反向引用、交/补等扩展；经典正则表达式匹配不是 NP-hard。
- 第 145 行 Myhill-Nerode 定理中“状态数最小”默认只考虑可达状态。建议在定理或证明前说明 DFA 已删除不可达状态。
- 第 150 行“每个等价类选择其中最短的任意一个字符串作为代表元”不是必要条件，可能让读者误以为最短性参与证明。可改为“任选一个代表元”。
- 第 197 行 Hopcroft 算法中“加入 $U,V$ 中的任意一个”对正确性可以，但复杂度分析依赖加入较小者。建议正文直接写“加入较小者”。

## `chapter4.tex`：Trie 和 AC 自动机

### 应修正

- 第 13 行“叶子结点对应的恰好是这些字符串”在某个字符串是另一个字符串前缀时不成立。应改为“终止标记结点对应这些字符串”。
- 第 24 行压缩 Trie 结点数“不超过 $2|L|$”需要说明保留终止结点，且最好先去重；否则前缀关系和重复字符串会让表述不清。
- 第 95 行两处 `\mathtt{fail}[mathrm{son}_{u,c}]` 少了反斜杠，应为 `\mathtt{fail}[\mathrm{son}_{u,c}]`。
- 第 110 行“模式串在 $T$ 中的总出现次数不超过 $2|T|\sqrt L$”需要假设模式串按内容去重；若有大量重复模式串，结论不成立。
- 第 229 行 $C_{i,j}(x)$ 中 `s_j[|s_j|-k+1,k]` 区间右端点应为 `|s_j|`，即后缀 $s_j[|s_j|-k+1,|s_j|]$。

### 建议澄清

- 第 24 行“直接实现空间复杂度与普通 Trie 相同”不一定成立：若边标签只存原串区间，压缩 Trie 空间可以随结点数下降。建议区分“复制边标签”和“存引用区间”两种实现。
- 第 92 行“AC 自动机无法支持在线插入”过于绝对。建议改为“静态 AC 自动机的 fail/转移不支持简单地逐串增量维护；动态集合通常用二进制分组/重构”。
- 第 95 行“只需要 fail 数组时复杂度与 $|\Sigma|$ 无关”的证明建议展开或给参考，因为朴素跳 fail 在错误实现下可能退化。
- 第 131 行公式前缺少逗号：“$v_k^{(j)}$ 那么相当于”建议改为“$v_k^{(j)}$，那么相当于”。
- 第 199--203 行禁止子串 DP 的 $f_u$、$g_u$ 是当前层还是下一层不够明确，建议先定义 DP 层转移含义。

## `chapter5.tex`：后缀结构

### 应修正

- 第 36 行“$s[k+1]$ 对应的位置”应为“后缀 $s[k+1:]$ 对应的位置”。
- 第 177 行 SAM 构造代码 `int v=a[u].ch[ch];` 应为 `int v=a[u].ch[c];`。
- 第 283 行 `\mathrm{LCP(s[j:],s[i:])}` 建议改为 `\mathrm{LCP}(s[j:],s[i:])`。

### 建议澄清

- 第 5 行“隐式后缀树/后缀树”的定义建议补充特殊终止符 `$` 的版本。很多资料默认后缀树是对 $s\$$ 的后缀建压缩 Trie，这能避免后缀停在边中间。
- 第 92 行“基于相同道理再写一遍差不多的代码”对读者帮助有限。建议说明加终止符是更常见、更安全的做法。
- 第 129--142 行 SAM 的 endpos 等价类建议补充“这里讨论的是 $s$ 的子串；若作为识别后缀的 DFA，需要把后缀对应状态设为接受态”。
- 第 191--197 行 SAM 转移边上界可保留，但建议注明常见精确上界是 $3n-4$（$n\ge 2$），当前 $3n-2$ 是较松上界。
- 第 204 行广义 SAM 代码需要说明 `cur` 在 Trie 多分支构造中如何保存/恢复，否则读者可能按单串写法误用。

## `chapter6.tex`：回文结构

### 应修正

- 第 10--12 行 Manacher 的半径下界疑似有一位偏差。按当前 $h_i$ 定义，常见初值应为 `min(h[i'], j+h[j]-i)`；正文写 `j+h_j-i-1` 虽然可能仍能扩展出正确值，但会使证明和复杂度说明不标准。
- 第 46 行 `\text{后一半}` 出现在普通文本模式中，不建议这样写。直接写“后一半”即可。
- 第 64--74 行回文树代码缺少两个根的初始化约定，长度为 $1$ 的新结点的 fail 特判也不明显。建议补完整初始化，或明确这是省略版伪代码。

### 建议澄清

- 第 46 行“接受所有回文子串的后一半的树形 DFA”不是回文树的常见定义，容易让读者困惑。建议改为“每个非根结点对应一个本质不同回文子串，边表示在两侧添加相同字符”。
- 第 56 行“偶根的 link 指向奇根”建议同时说明奇根的 link 指向自己，以及二者长度分别为 $0,-1$。
- 第 79--82 行线性复杂度证明建议补充势能：每次沿 link 上跳都会降低当前候选回文长度，总下降次数被总增长次数控制。
- 第 138 行 $p_u=\mathrm{len}(u)-\mathrm{len}(\mathtt{link}[u])$ 对应最小周期的结论建议给一行证明或引用 Border 与周期章节。
